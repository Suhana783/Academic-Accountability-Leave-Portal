1. Only admins create user accounts (no public signup)
2. Role is stored in database and server-determined
3. Login is simple: email + password (no role selector)
4. After login, user is auto-redirected based on their role
5. Token persists across page refreshes
6. On token expiration, user is auto-logged out
7. Both frontend and backend validate permissions

This matches how real college/institution systems work and is much
more secure than allowing self-signup."
